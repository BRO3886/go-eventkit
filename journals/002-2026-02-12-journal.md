- **CGColor not NSColor**: Calendar/list color uses `CGColorRef` via `CGColorCreateGenericRGB()` â€” avoids AppKit dependency for the color path
- **Array-wrapping pattern**: Single JSON object responses reuse existing `parseListsJSON`/`parseCalendarsJSON` by wrapping in `"[" + jsonResp + "]"` â€” no need for separate single-object parsers

- **Immutability check**: `cal.isImmutable` must be checked before update/delete â€” Apple returns generic error otherwise. We check explicitly and return `ErrImmutable`.

## What Failed and Why
1. **`defaultCalendarForNewEvents.source` rejected calendar creation**: The default calendar was on Google CalDAV, which allows event CRUD but rejects `saveCalendar:commit:` (calendar creation). Error: "That account does not allow calendars to be added or removed."
2. **iCloud source name collision**: `store.sources` contains separate EKSource objects with identical title "iCloud" â€” one for events, one for reminders. Naive `find_source_by_name` returned the wrong one. Calendar creation got the reminders-iCloud source â†’ "That account does not support events."
3. **Iterating sources by sourceType didn't help**: CalDAV + iCloud title match still hit the wrong source. Only `[source calendarsForEntityType:]` reliably distinguishes them.

## Key Insights
- **macOS has duplicate "iCloud" EKSource objects**: One supports `EKEntityTypeEvent`, the other `EKEntityTypeReminder`. They share the same `.title` but are different objects with different `sourceIdentifier` values.
- **`find_source_by_name` must be entity-aware**: Use `[source calendarsForEntityType:]` to prefer the correct source:
  ```objc
  // In calendar package:
  NSSet* eventCals = [source calendarsForEntityType:EKEntityTypeEvent];
  if (eventCals.count > 0) return source;
  // In reminders package:
  NSSet* remCals = [source calendarsForEntityType:EKEntityTypeReminder];
  if (remCals.count > 0) return source;
  ```
- **No EventKit API to check if a source supports calendar creation**: `isImmutable` on calendars checks write access for events/reminders, not for calendar container creation. Google CalDAV has writable calendars but rejects new calendar creation.
- **Making Source required is the right call**: Auto-detecting the correct source is fragile. Users know their source names from `Calendars()`/`Lists()` output.

## Decisions Made
- **Source is required for create operations**: Removes all ambiguity. Consumer code calls `Calendars()`/`Lists()` first to discover source names. This is explicit and predictable.
- **Entity-aware source lookup with fallback**: `find_source_by_name` prefers entity-matching source, falls back to any name match if no entity match exists. Handles edge cases without breaking existing lookups.
